# Windows 项目运行操作文档设计

## 目标

在项目根目录新增 `WINDOWS_SETUP_GUIDE.md`，指导完全没有前后端开发经验的用户在 Windows 10/11 上，从拿到项目文件开始，独立启动并验证 LeetCode Copilot。

## 读者与前置条件

- 读者不了解前后端分离项目的运行方式。
- 电脑已经安装 Python、Node.js 和 npm。
- 文档只说明如何运行本项目，不包含 Python、Node.js、npm 或编辑器的安装教程。
- 命令统一使用 Windows PowerShell。

## 文档结构

1. 简要说明项目由 FastAPI 后端和 React/Vite 前端组成，运行时需要保持两个 PowerShell 窗口。
2. 说明如何进入解压后的项目根目录。
3. 后端流程：进入 `backend`、创建并激活虚拟环境、安装依赖、复制环境变量文件、启动服务。
4. 后端验证：访问健康检查和 Swagger 接口文档。
5. 前端流程：另开 PowerShell、进入 `frontend`、安装依赖、启动服务。
6. 完整验证：访问前端页面，提交测试题目，确认 Mock 模式能够生成结果并保存记录。
7. 可选 DeepSeek 配置：填写 `backend/.env`，重启后端后使用真实模型。
8. 说明如何停止服务，以及下次启动时哪些安装步骤无需重复。
9. 给出初学者最常遇到的故障及对应处理方法。

## 命令与表述规范

- 每组命令前明确当前所在目录。
- 命令保持可直接复制，使用项目根目录占位符而不是假设固定盘符。
- 每个关键步骤说明成功时应看到的端口、页面或终端信息。
- 清楚区分“首次运行”和“以后再次运行”。
- 不要求用户配置 DeepSeek API Key；未配置时使用 Mock Provider 完成基础验证。
- API Key 只写入 `backend/.env`，并明确禁止提交真实 Key。

## 验证标准

- 后端 `http://127.0.0.1:8000/health` 能够返回成功响应。
- 前端 `http://localhost:5173` 能够打开。
- 用户可进入解题页面，提交不少于 10 个字符的题目文本，并看到 Mock 解析结果。
- 文档中的构建和测试命令与当前项目脚本、依赖文件及端口配置一致。

## 边界

- 不修改应用代码、依赖或启动方式。
- 不介绍部署到服务器、Docker、数据库管理或编辑器安装。
- 不把未跟踪的 `deepresearch` 目录纳入 LeetCode Copilot 的运行流程。
