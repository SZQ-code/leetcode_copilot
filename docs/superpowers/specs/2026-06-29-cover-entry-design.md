# 蓝色独石封面入口设计

- 日期：2026-06-29
- 状态：已确认
- 范围：前端根路由、封面组件及进入工作区的过渡
- 视觉方向：C 方案，蓝色独石
- 过渡方向：C 方案，深度推进
- 路由方案：独立封面路由

## 背景

LeetCode Copilot 当前在访问根网址时直接显示学习工作区。工作区适合持续操作，但缺少独立的第一视觉层，用户首次打开项目时无法先感知产品名称、定位和品牌氛围。

本次新增一个独立全屏封面。用户访问根网址时先看到简约深蓝科技风格的品牌页面，点击任意位置后再进入现有工作区。

## 目标

1. 根网址提供明确、简洁、有科技感的产品第一印象。
2. 保持现有深色 IDE 工作区不变。
3. 点击封面任意位置即可进入工作区。
4. 支持键盘和减少动态效果设置。
5. 深层链接继续直接进入目标工作区页面。
6. 封面不提前请求后端健康状态或学习数据。

## 非目标

- 不新增登录、注册、引导流程或轮播内容。
- 不增加图片、视频、粒子系统或 WebGL。
- 不记录用户是否已经看过封面。
- 不让所有路由强制经过封面。
- 不修改后端 API、数据库或业务逻辑。
- 不修改现有工作区页面的视觉设计。

## 已选方案

### 视觉：蓝色独石

采用极简纯深蓝背景、超大空心 `LC` 轮廓字和单条扫描线。

未采用的视觉方向：

1. 深海信号：品牌一致性强，但圆环和网格装饰多于用户选择的极简方向。
2. 数据地平线：展示感强，但代码状态流会削弱封面的简洁程度。

### 过渡：深度推进

用户触发进入操作后，封面向前放大并淡出，随后进入工作区。

未采用的过渡方向：

1. 瞬时切换：速度最快，但会中断封面的仪式感。
2. 扫描淡出：节制且清晰，但用户选择了更有纵深感的推进方式。

### 路由：独立路由

封面和工作区首页使用不同路由，避免封面状态与 `AppShell` 耦合。

未采用的路由方向：

1. 同路由状态切换：URL 不变化，但刷新和“首页”语义不清晰。
2. 工作区上方覆盖封面：过渡顺滑，但会提前加载工作区和后端数据。

## 路由结构

```text
/                  -> CoverPage
/workspace         -> AppShell + HomePage
/solve             -> AppShell + SolvePage
/history           -> AppShell + HistoryPage
/categories        -> AppShell + CategoryPage
/problems/:id      -> AppShell + ProblemDetailPage
/*                 -> AppShell + NotFoundPage
```

规则：

1. 每次访问或刷新 `/` 都显示封面。
2. 点击封面后使用普通客户端导航进入 `/workspace`。
3. 浏览器返回 `/` 时重新显示封面。
4. 直接访问 `/workspace` 或其他深层链接时不显示封面。
5. `AppShell` 中的 Logo 和“首页”导航统一指向 `/workspace`。
6. 未知路由继续由工作区内的 404 页面处理。

## 组件边界

### `CoverPage`

职责：

- 渲染封面文字和纯 CSS 背景。
- 接收鼠标、触摸和键盘进入操作。
- 管理 `idle` 与 `exiting` 两个本地状态。
- 在退出动画结束后导航到 `/workspace`。

依赖：

- React 本地状态。
- React Router 的 `useNavigate`。
- 独立的 `CoverPage.css`。

不负责：

- 加载后端健康状态。
- 加载学习记录或题型统计。
- 渲染工作区导航。
- 持久化“已看过封面”状态。

### `AppShell`

继续负责工作区导航和后端连接状态，但不包裹 `CoverPage`。

需要调整：

- Logo 链接由 `/` 改为 `/workspace`。
- “首页”导航由 `/` 改为 `/workspace`。
- 路由上下文将 `/workspace` 识别为学习工作区首页。

### `App`

将路由拆分为：

- 独立的根路由封面；
- 由 `AppShell` 包裹的工作区路由集合。

封面和工作区不共享进入状态。

## 封面内容

页面中央依次展示：

```text
ALGORITHM LEARNING SYSTEM

LEETCODE COPILOT

让每一道题，沉淀成可复用的方法。
```

页面下方展示：

```text
点击任意位置开始学习
```

界面不增加功能介绍、技术标签、按钮组或其他说明文字。

## 视觉系统

### 颜色

| Token | 色值 | 用途 |
| --- | --- | --- |
| `cover-void` | `#071120` | 主背景 |
| `cover-deep` | `#06101E` | 深层背景 |
| `cover-line` | `rgba(109, 178, 242, 0.10)` | `LC` 轮廓与辅助线 |
| `cover-accent` | `#68BBF1` | 小标题和扫描线 |
| `cover-text` | `#F3F7FF` | 主标题 |
| `cover-muted` | `#879BB7` | 定位文案 |

### 字体

- 主标题使用现有 `Bahnschrift Condensed` 字体栈。
- 定位文案使用现有正文 `Segoe UI Variable` 字体栈。
- 系统标签和点击提示使用现有 `Cascadia Code` 字体栈。
- 不加载远程字体。

### 构图

- 页面占满可视区域，最小高度使用动态视口单位。
- 超大空心 `LC` 位于背景中央，不抢夺主标题可读性。
- 项目名称和定位文案垂直居中。
- 点击提示位于中下部，并保留移动端安全距离。
- 页面不出现滚动条或横向溢出。

## 交互与动效

### 进入触发

整个封面拥有一个覆盖全屏的透明交互层：

- 鼠标点击任意位置；
- 触摸任意位置；
- 键盘聚焦后按 `Enter`；
- 键盘聚焦后按空格。

交互层提供明确的可访问名称“进入学习工作区”。

### 空闲状态

- 一条低亮度蓝色扫描线缓慢横向移动。
- 扫描线不持续闪烁，不影响文字阅读。
- 点击提示可以使用轻微透明度呼吸，但不缩放。

### 退出状态

触发后：

1. 禁止重复触发。
2. 主体从 `scale(1)` 推进到约 `scale(1.18)`。
3. 背景和文字同步淡出。
4. 动画持续约 `800ms`。
5. 动画结束后导航到 `/workspace`。

若动画结束事件未触发，使用同等时长的兜底计时完成导航。

### 减少动态效果

当 `prefers-reduced-motion: reduce` 生效时：

- 停止扫描线和呼吸动画。
- 退出只进行极短淡出。
- 在不超过 `100ms` 内进入 `/workspace`。

## 响应式设计

### 桌面端

- 主标题保持单行或自然两行。
- 空心 `LC` 轮廓占据背景主要区域。
- 点击提示位于页面中下部。

### 移动端

- 主标题按屏幕宽度缩放。
- 空心 `LC` 轮廓缩小，不能造成横向溢出。
- 定位文案限制为两行以内。
- 点击提示避开浏览器底部安全区域。
- 整页保持单屏可见，不依赖滚动。

## 数据流与错误处理

封面没有远程数据请求。

本地状态流：

```text
idle
  |
  | click / Enter / Space
  v
exiting
  |
  | animation end or timeout
  v
navigate("/workspace")
```

导航是唯一可能失败的操作。React Router 客户端导航不可用时，现有应用本身也无法正常运行，因此本阶段不增加独立错误界面。

## 可访问性

- 全屏交互层可通过键盘聚焦。
- 使用语义化按钮或等效可访问交互层。
- 提供可见的 `:focus-visible` 状态。
- 不依赖扫描线表达可操作性。
- 点击提示明确说明操作结果。
- 文字与背景对比度满足 WCAG AA。
- 减少动态效果时停止非必要动画。

- `git diff --check` 成功。

### 浏览器验证

10. 页面无控制台错误、横向溢出或垂直滚动。
